\# Robot Journal 1.0



\## Цель проекта



Создать максимально простой Telegram-бот для учёта работы роботизированного участка.



Главная задача — заменить бумажный журнал и обеспечить передачу информации между операторами и инженером.



\---



\# Пользователи



\## Главный оператор



Егор



Ярослав



Игорь



Работают по Telegram ID.



Ведут учёт трёх станций.



\---



\## Инженер



Игорь



Получает отчёт после завершения работы оператором.



Может получить отчёт по номеру рамы.



\---



\# Роботизированный участок



Количество станций:



3



На каждой станции одновременно находится только одна рама.



\---



\# Жизненный цикл рамы



📦 Загрузили



↓



▶ Начало сварки



↓



⏹ Окончание сварки



↓



📤 Выгрузили



↓



Станция свободна



\---



\# Главное правило интерфейса



На экране всегда отображается только следующее возможное действие.



Никаких лишних кнопок.



\---



\# Переходящие рамы



Рама не принадлежит смене.



Рама принадлежит станции.



Следующий оператор продолжает работу.



\---



\# Меню оператора



🏭 Станция 1



🏭 Станция 2



🏭 Станция 3



📊 Состояние участка



📋 Отчёт



🏁 Завершить работу



\---



\# Меню инженера



📊 Состояние участка



📋 Отчёт по раме



\---



\# Отчёт оператора



После завершения работы автоматически отправляется инженеру.



Отчёт содержит:



• завершённые рамы



• переходящие рамы



• время между этапами



• полный цикл каждой завершённой рамы
